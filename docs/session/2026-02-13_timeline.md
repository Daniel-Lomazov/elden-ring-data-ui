## Phase 4 — Recurring interactive failure (`enter`)

### User meaning
- Interactive mode repeatedly failed with: `Histogram render error: enter`.
- Expected a root-cause fix, not repeated temporary patches.

### Deep cause analysis
- Interactive-only path passed `st` (module) as target.
- Renderer unconditionally used `with target_ui:`.
- `st` is not a context manager, producing `__enter__/enter` failure.

### Changes made
- Updated interactive renderer to use context manager only when supported.
- Replaced module-target usage with direct target methods (`target_ui.info/warning/plotly_chart`).
- Kept safe fallback behavior to avoid crashing entire histogram block.

### Areas touched
- `app.py`

### Outcome
- Specific `enter` crash path removed at source.
